1. Project Overview

What is Panchayat?

Panchayat is a comprehensive web-based Housing Society Management System built specifically for Indian residential societies. It replaces outdated WhatsApp groups, paper registers, and manual processes with a modern AI-powered platform that handles complaint management, bylaw lookups, service bookings, and transparent maintenance accounting.

Target Audience
Societies with 50 to 500+ flats, managed by housing society committees in India.

The Problem It Solves

  • 420 unread WhatsApp messages nobody responds to
  • 10-minute voice notes about leaky taps going untracked
  • Residents not knowing where their maintenance money goes
  • Parking disputes at 11 PM with no resolution system
  • Dusty rule books nobody reads or knows how to find
  • Domestic staff entering without verification
  • AGM chaos because accounts are not transparent

The Solution

  • Centralized complaint ticketing with voice support
  • AI that reads complaints and gives a 3-point summary
  • Bylaw bot that answers "Can I renovate on Sunday?" in seconds
  • Transparent maintenance ledger with plain-language AI explanation
  • Service booking for plumber, electrician, laundry, carpenter
  • Role-based access for Admin, Committee, and Residents

2. User Roles & Permissions

Three User Roles

Admin Super User (Society Admin)

Full system access across all societies. Can manage societies, users, roles, bylaws, services, and view audit logs.

Access URL: /admin-panel/

Committee Committee Member (Secretary/Treasurer)

Society-level access. Manage complaints, post notices, track dues, and view service bookings.

Access URL: /committee/

Resident Flat Owner/Tenant

Personal access only. Submit complaints, book services, view dues, and read notices.

3. Authentication System

Login Page

Access the login page at:

http://127.0.0.1:8000/login/

Enter your email and password. On success:

  • JWT token stored in localStorage
  • Role stored in localStorage
  • Redirect based on role:
    • Admin → /admin-panel/
    • Committee → /committee/
    • Resident → /resident/

Registration

New residents can register at:

http://127.0.0.1:8000/register/

After submission, the account is pending approval. An admin must approve the user before they can login.

Note
New registrations require admin approval before the user can login. Until approved, login attempts return "Account not approved yet".

3B. Society Setup Workflow

New Workflow: Society Activation

The system now implements a controlled onboarding flow to ensure proper society initialization:

  1. Admin logs in and seeds the society (via management command)
  2. Society is created with is_active: False — residents cannot be added yet
  3. Admin assigns committee members (Secretary & Treasurer) via Admin Panel
  4. Society automatically becomes is_active: True
  5. Secretary can now add residents to the society
Why This Workflow?
This ensures only legitimate committee members can add residents, preventing unauthorized registrations and maintaining proper society governance.

Assign Committee Members (Admin)

Endpoint: POST /api/auth/assign-committee/

Request body:

{
  "society_id": 1,
  "secretary": {
    "email": "secretary@splendour.com",
    "name": "John Secretary",
    "mobile": "9876543210",
    "password": "Committee@123"
  },
  "treasurer": {
    "email": "treasurer@splendour.com", 
    "name": "Jane Treasurer",
    "mobile": "9876543211",
    "password": "Committee@123"
  }
}

On success:

  • Secretary and Treasurer users created
  • CommitteeMember records created
  • Society is_active set to True
  • Audit log entries created

Add Resident (Secretary Only)

Endpoint: POST /api/auth/add-resident/

Only the Secretary can add residents. Public registration is disabled.

{
  "email": "resident@example.com",
  "name": "New Resident",
  "mobile_no": "9876543212",
  "password": "Resident@123",
  "flat_no": "101",
  "wing_no": "A"
}
Key Changes
  • Public /register/ page is disabled — only Secretary can add residents
  • Residents added by Secretary are auto-approved
  • Society must be active before residents can be added

4. Complaint Management

4.1 Submit Text Complaint (Resident)

Residents can submit text complaints with the following fields:

  • Title - Brief description (required)
  • Description - Full details (optional)
  • Category - Plumbing, Electrical, Lift, Parking, Noise, Cleanliness, Security, Other
  • Priority - Low, Medium, Urgent

Auto-assigns to the resident's society with status Open.

4.2 Submit Voice Complaint (Resident)

Voice complaints enable residents to speak in Hindi, Marathi, English, or Tamil:

  1. Click the microphone button
  2. Allow microphone permission when browser asks
  3. Speak your complaint
  4. Click Stop when done
  5. Wait for AI transcription (3-5 seconds)
  6. Review and edit the transcript if needed
  7. Submit the complaint

Supported formats: WebM, WAV, MP3, MP4, OGG, FLAC. Max size: 10MB.

4.3 View My Complaints (Resident)

Shows all complaints submitted by the resident with:

  • Title, status badge, priority badge, category, date
  • Click to expand: full description, timeline, assigned to, notes
  • Status timeline: Open → In Progress → Resolved → Closed

4.4 Edit Complaint

Edit button visibility rules:

  • YES Edit = Open or In Progress
  • NO Edit = Resolved or Closed

4.5 Delete Complaint

Delete button visibility rules:

  • YES Delete = Open only
  • NO Delete = In Progress, Resolved, Closed

4.6 Manage All Complaints (Committee)

Committee members can:

  • View all complaints with filters (Status, Priority, Category)
  • Assign to committee members or vendors
  • Change status: Open → In Progress → Resolved
  • Add internal notes

4.7 AI Inbox Summary (Committee/Admin)

Auto-loads on dashboard. Fetches today's complaints and sends to Gemini API for summarization:

  • Groups similar complaints
  • Flags urgent with [URGENT]
  • Shows total count and unique flats
  • Cached for 5 minutes
  • Refresh button available

5. Bylaw Bot (AI Feature)

5.1 Upload Bylaw PDF (Admin)

  1. Go to Admin Panel → Bylaws
  2. Click "Upload PDF"
  3. Enter document title and version
  4. Select PDF file (max 10MB)
  5. Upload → text automatically extracted
  6. Available for AI queries immediately

5.2 Ask Bylaw Bot

Chat-style interface with quick-question chips:

  • "Gym hours?"
  • "Can I renovate on Sunday?"
  • "Visitor parking rule?"

Or type any question. Gemini reads bylaw text and answers, citing exact Rule numbers.

Sample Questions
  • Can I renovate on Sunday? → Rule 4.2: Not permitted...
  • What are gym hours? → Rule 14.1: 6 AM to 10 PM...
  • Can I keep a pet? → Rule 8.1: NOC required...

6. Service Booking System

6.1 Available Services

Default services seeded in the system:

  • Plumber - Fix taps, pipes, drainage
  • Electrician - Wiring, switches, MCB
  • Laundry - Pickup and delivery
  • Carpenter - Furniture and fixtures

6.2 How to Book (Resident)

  1. Go to Services tab
  2. See service cards with vendor name and price
  3. Click "Book" on desired service
  4. Select date from date picker
  5. Available time slots load as radio buttons
  6. Select slot and click Confirm
  7. Booking confirmed with ID
  8. Slot marked unavailable for others

6.3 Cancel Booking

My Bookings section shows all bookings. Cancel available if more than 2 hours before slot. On cancel, slot becomes available again.

6.4 Admin Manages Services

Admin can add new services with vendor details, set price per slot, generate slots in bulk, and activate/deactivate services.

7. Maintenance & Accounts

7.1 Monthly Maintenance Ledger (Committee)

Committee enters expense categories monthly:

  • Staff salaries
  • Lift AMC
  • Generator fuel
  • Water charges
  • Sinking fund
  • Garden and misc

Total auto-calculated. AI generates plain-language explanation.

7.2 Dues Management (Committee)

When maintenance is saved, dues auto-generated for all flats using formula:

per_flat_amount = total_maintenance / total_flats

Committee can mark dues as paid with payment reference number.

7.3 Resident Dues View

Residents see current month dues with paid/unpaid status, AI explanation, and payment history.

8. Notice Board

8.1 Post Notice (Committee/Admin)

Fields: Title, Body, Pin toggle, Expiry date. Pinned notices appear first always.

8.2 View Notices (All Roles)

Pinned notices shown at top with pin icon. All notices in reverse chronological order.

8.3 Delete Notice

Delete button on each notice with confirmation.

9. Admin Panel Features

9.1 Society Management

Add, edit, activate/deactivate societies. View stats: total flats, users, open complaints.

9.2 User Management

View all users with role filter. Approve pending registrations. Change roles. Deactivate accounts.

9.3 Bylaw PDF Management

Upload new PDF, view version history, set active version, preview extracted text.

9.4 Service & Vendor Management

Add/edit services, manage vendor details, generate time slots in bulk.

9.5 Audit Log

Every action logged: who, what, when, which record. Filterable and paginated. Cannot be deleted or modified.

12. How to Install

-

Prerequisites

  • Python 3.11+ → python.org/downloads
  • MySQL 8.0+ → dev.mysql.com
  • Git (optional) → git-scm.com

Step 1: Create Project

git clone https://github.com/Vikash97600/Panchayat-housing-society.git
cd Panchayat-housing-society-main

Step 2: Create Virtual Environment

python -m venv venv

# Windows
venv\Scripts\activate

# Mac/Linux
source venv/bin/activate

Step 3: Install Dependencies

pip install -r requirements.txt

Step 4: Create MySQL Database

CREATE DATABASE panchayat_db 
  CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;

Step 5: Create .env File

SECRET_KEY=your-django-secret-key
DEBUG=True
DB_NAME=panchayat_db
DB_USER=root
DB_PASSWORD=root
DB_HOST=localhost
DB_PORT=3306
GEMINI_API_KEY=your-gemini-api-key
GROQ_API_KEY=your-groq-api-key
Getting API Keys
Gemini: aistudio.google.com → Sign in → Get API Key
Groq: console.groq.com → Sign up → API Keys

Step 6: Run Migrations

python manage.py migrate

Step 7: Seed Sample Data

python manage.py seed_panchayat

Step 8: Run the Server

python manage.py runserver

# Open browser
http://127.0.0.1:8000

14. Troubleshooting

Installation Errors

ERROR: mysqlclient not found
Fix:
  • Windows: pip install mysqlclient
  • Mac: brew install mysql-client then pip install mysqlclient
  • Ubuntu: sudo apt install python3-dev default-libmysqlclient-dev then pip install mysqlclient
ERROR: django.db.utils.OperationalError: (1049) Unknown database
Fix: Create database first in MySQL: 
CREATE DATABASE panchayat_db CHARACTER SET utf8mb4;

Login Errors

ERROR: Invalid credentials
Fix: Run seed command again: 
python manage.py seed_panchayat
ERROR: Redirects to login on every page
Fix: Token expired. Login again. Check localStorage in DevTools (F12) → Application → Local Storage.

Voice Recording Errors

ERROR: Microphone permission denied
Fix: Click lock icon in browser URL bar → Allow Microphone. Voice only works on localhost or HTTPS.

Bylaw Bot Errors

ERROR: Bot says "Bylaws not found"
Fix: Admin must upload bylaw PDF first. Go to /admin-panel/ → Bylaws → Upload PDF.

Edit/Delete Button Not Showing

Edit or Delete button missing on complaint
Check:
  • Edit shows for: Open, In Progress
  • Delete shows for: Open only
  • Open DevTools → Network → /api/complaints/ → check can_edit/can_delete in response
